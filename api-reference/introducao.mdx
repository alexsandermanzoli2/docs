## Base URL

All API requests should be made to:

```
https://api.speedio.com.br/v1
```

For sandbox/testing:

```
https://sandbox-api.speedio.com.br/v1
```

## Authentication

All requests must include your API key in the Authorization header:

```bash
Authorization: Bearer YOUR_API_KEY
```

## Request Format

- All requests must use **HTTPS**
- Request bodies must be **JSON** with `Content-Type: application/json`
- Responses are returned as **JSON**

## Pagination

List endpoints support pagination using these parameters:

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `page` | integer | 1 | Page number |
| `limit` | integer | 20 | Items per page (max: 100) |

### Example Request

```bash
GET /v1/leads?page=2&limit=50
```

### Pagination Response

```json
{
  "data": [...],
  "pagination": {
    "page": 2,
    "limit": 50,
    "total": 234,
    "pages": 5,
    "has_more": true
  }
}
```

## Filtering

Use query parameters to filter results:

```bash
GET /v1/leads?status=active&created_after=2024-01-01
```

Common filters:

- `status` - Filter by status
- `created_after` - Filter by creation date (ISO 8601)
- `created_before` - Filter by creation date (ISO 8601)
- `updated_after` - Filter by update date (ISO 8601)
- `updated_before` - Filter by update date (ISO 8601)

## Sorting

Use the `sort` parameter to sort results:

```bash
GET /v1/leads?sort=-created_at
```

- Prefix with `-` for descending order
- Default: `-created_at` (newest first)

## Field Selection

Use the `fields` parameter to select specific fields:

```bash
GET /v1/leads?fields=id,email,name,company
```

## Batch Operations

Some endpoints support batch operations:

```json
POST /v1/leads/batch
{
  "operations": [
    {
      "method": "create",
      "data": { ... }
    },
    {
      "method": "update",
      "id": "lead_123",
      "data": { ... }
    }
  ]
}
```

## API Versioning

The API version is included in the URL path:

- Current version: `v1`
- Version in URL: `https://api.speedio.com.br/v1/...`

<Warning>
  We'll notify you at least 6 months before deprecating any API version.
</Warning>

## HTTP Methods

| Method | Usage |
|--------|-------|
| `GET` | Retrieve resources |
| `POST` | Create new resources |
| `PUT` | Update entire resources |
| `PATCH` | Partial resource updates |
| `DELETE` | Delete resources |

## Status Codes

| Code | Description |
|------|-------------|
| `200 OK` | Success |
| `201 Created` | Resource created |
| `204 No Content` | Success with no response body |
| `400 Bad Request` | Invalid request |
| `401 Unauthorized` | Invalid or missing API key |
| `403 Forbidden` | Insufficient permissions |
| `404 Not Found` | Resource not found |
| `422 Unprocessable Entity` | Validation error |
| `429 Too Many Requests` | Rate limit exceeded |
| `500 Internal Server Error` | Server error |

## Data Types

### Dates

All dates are in ISO 8601 format with timezone:

```
2024-01-15T10:30:00Z
```

### Phone Numbers

Phone numbers should include country code:

```
+55 11 98765-4321
```

### Money

Monetary values are in cents (integers):

```json
{
  "amount": 10000, // R$ 100.00
  "currency": "BRL"
}
```
