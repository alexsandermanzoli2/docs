## Visão Geral

O endpoint de enriquecimento permite adicionar informações detalhadas a seus dados existentes. Você pode enriquecer dados de empresas ou pessoas usando diferentes tipos de entrada como e-mail, telefone, CNPJ ou nome da empresa.

## Requisição

### URL
```
POST https://api-get-leads.speedio.com.br/enrich
```

### Request Body

```python Python
import requests
import base64
import json
import os

def enrich_data(data_type, data_list, **options):
    """
    Enriquecer dados usando a API Speedio
    """
    # Configure as variáveis de ambiente
    username = os.getenv('SPEEDIO_USERNAME')
    password = os.getenv('SPEEDIO_PASSWORD')
    
    if not username or not password:
        raise ValueError('SPEEDIO_USERNAME e SPEEDIO_PASSWORD devem estar definidas')
    
    auth = base64.b64encode(f"{username}:{password}".encode()).decode()
    
    payload = {
        'type': data_type,
        'data': data_list,
        **options
    }
    
    headers = {
        'Content-Type': 'application/json',
        'Authorization': f'Basic {auth}'
    }
    
    response = requests.post(
        'https://api-get-leads.speedio.com.br/enrich',
        json=payload,
        headers=headers
    )
    
    if response.status_code == 200:
        result = response.json()
        print(f"{result['metadata']['total_enriched']} perfis enriquecidos")
        return result['enriched']
    else:
        print(f"Erro: {response.status_code}")
        print(response.text)
        return None

# Enriquecer por telefone
phones = ['+5511987654321', '+5521976543210']
enriched = enrich_data('phone', phones, include_company=True, verify_data=True)

# Enriquecer por empresa
companies = ['Microsoft Brasil', 'Magazine Luiza']
enriched = enrich_data('company', companies, include_technologies=True)
```

<?php
function enrichData($type, $data, $options = []) {
    // Configure as variáveis de ambiente
    $username = $_ENV['SPEEDIO_USERNAME'] ?? getenv('SPEEDIO_USERNAME');
    $password = $_ENV['SPEEDIO_PASSWORD'] ?? getenv('SPEEDIO_PASSWORD');
    
    if (!$username || !$password) {
        throw new Exception('SPEEDIO_USERNAME e SPEEDIO_PASSWORD devem estar definidas');
    }
    
    $auth = base64_encode($username . ':' . $password);
    
    $payload = array_merge([
        'type' => $type,
        'data' => $data
    ], $options);
    
    $curl = curl_init();
    
    curl_setopt_array($curl, [
        CURLOPT_URL => 'https://api-get-leads.speedio.com.br/enrich',
        CURLOPT_RETURNTRANSFER => true,
        CURLOPT_POST => true,
        CURLOPT_POSTFIELDS => json_encode($payload),
        CURLOPT_HTTPHEADER => [
            'Content-Type: application/json',
            'Authorization: Basic ' . $auth
        ]
    ]);
    
    $response = curl_exec($curl);
    $httpCode = curl_getinfo($curl, CURLINFO_HTTP_CODE);
    curl_close($curl);
    
    if ($httpCode == 200) {
        $result = json_decode($response, true);
        echo $result['metadata']['total_enriched'] . " perfis enriquecidos\n";
        return $result['enriched'];
    } else {
        echo "Erro: HTTP " . $httpCode . "\n";
        echo $response;
        return null;
    }
}

// Enriquecer por LinkedIn
$linkedinUrls = [
    'https://linkedin.com/in/joaosilva',
    'https://linkedin.com/company/empresa'
];

$enriched = enrichData('linkedin', $linkedinUrls, [
    'include_company' => true,
    'include_social' => true
]);
?>

## Tipos de Enriquecimento

### 1. Por E-mail
Ideal para enriquecer listas de contatos existentes:

```json
{
  "type": "email",
  "data": ["contato@empresa.com"],
  "include_company": true,
  "verify_data": true
}
```

### 2. Por Telefone
Útil para validar e enriquecer números de telefone:

```json
{
  "type": "phone",
  "data": ["+5511987654321"],
  "include_company": true
}
```

### 3. Por CNPJ
Perfeito para pesquisa de empresas:

```json
{
  "type": "cnpj",
  "data": ["12345678000190"],
  "include_technologies": true,
  "include_social": true
}
```

### 4. Por Nome da Empresa
Quando você tem apenas o nome da empresa:

```json
{
  "type": "company",
  "data": ["Microsoft Brasil"],
  "include_technologies": true
}
```

### 5. Por LinkedIn
Para enriquecer perfis do LinkedIn:

```json
{
  "type": "linkedin",
  "data": ["https://linkedin.com/in/profile"],
  "include_company": true
}
```

## Casos de Uso

### 1. Enriquecimento de Lista de E-mails

```javascript
// Enriquecer uma lista de e-mails de prospects
const emails = [
  'joao@empresa1.com',
  'maria@empresa2.com',
  'carlos@empresa3.com'
];

const enriched = await enrichByEmail(emails, {
  include_company: true,
  verify_data: true
});

// Filtrar apenas e-mails válidos
const validEmails = enriched.filter(item => 
  item.verification.email.valid && 
  item.verification.email.deliverable
);
```

### 2. Pesquisa de Concorrentes

```python
# Enriquecer informações de empresas concorrentes
concorrentes = ['Empresa A', 'Empresa B', 'Empresa C']

enriched = enrich_data('company', concorrentes, 
    include_technologies=True,
    include_social=True
)

# Analisar tecnologias utilizadas
for company in enriched:
    if company['found']:
        print(f"{company['company']['name']}: {company['company']['technologies']}")
```

### 3. Validação de Base de Dados

```php
// Validar e enriquecer base de telefones
$phones = ['+5511987654321', '+5521976543210', '+5531987654321'];

$enriched = enrichData('phone', $phones, [
    'verify_data' => true,
    'include_company' => true
]);

// Separar números válidos e inválidos
$validPhones = array_filter($enriched, function($item) {
    return $item['verification']['phone']['valid'];
});
```

## Limites e Custos

| Tipo | Custo por Item | Máx. por Requisição |
|------|---------------|-------------------|
| Email | 1 crédito | 100 |
| Phone | 0.5 créditos | 100 |
| CNPJ | 1 crédito | 50 |
| Company | 1 crédito | 50 |
| LinkedIn | 1.5 créditos | 50 |

## Qualidade dos Dados

### Níveis de Confiança

- **0.9 - 1.0**: Dados altamente confiáveis
- **0.7 - 0.9**: Boa confiabilidade
- **0.5 - 0.7**: Confiabilidade moderada
- **< 0.5**: Baixa confiabilidade

### Freshness dos Dados

- **< 7 dias**: Dados muito frescos
- **< 30 dias**: Dados frescos
- **< 90 dias**: Dados atuais
- **> 90 dias**: Dados podem estar desatualizados
